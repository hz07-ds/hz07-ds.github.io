常见保护机制详解（Pwn 学习笔记）

发表于 2026-04-16 更新于 2026-04-20 分类于 网络安全 ， 学习笔记

一、保护机制总览表

保护 作用 影响 典型绕过 原理简述 检测命令
NX 栈 / 堆 不可执行 ❌ 无法直接执行 shellcode ret2libc、ROP、JOP 页表标志位 PTE_X 缺失，CPU 无法执行代码段 readelf -l ./bin | grep STACK
Canary 栈溢出检测 ❌ 覆盖返回地址前会触发 __stack_chk_fail 泄露 Canary、覆盖 TCB 中的值、One-by-one 爆破 在 ebp/rbp 与返回地址之间插入随机值，函数返回前校验 checksec --file=./bin 或 objdump -d
PIE 程序代码/数据段随机 ❌ 无法直接使用硬编码 gadget 地址 泄露任意代码地址（如 __libc_start_main） 加载时对 .text、.rodata、.got.plt 等段基址加偏移 readelf -h ./bin | grep Type (ET_DYN)
ASLR 内核级地址随机 ❌ libc / mmap / 栈 / 堆 基址变化 泄露 libc 地址（GOT / 函数地址） 内核随机化 mmap_base、stack_top、brk 起始位置 cat /proc/sys/kernel/randomize_va_space
RELRO GOT 表保护 ❌ Full RELRO 时无法修改 GOT ROP、hook __free_hook / _IO_file_jumps Partial：只保护 .got 不可写；Full：整个 GOT 只读并重定位 checksec 显示 Partial / Full

二、各保护细节与绕过

1. NX

  • 检测：readelf -l ./pwn | grep GNU_STACK，若结果中 RWE 没有 E 代表开启 NX。
  • 绕过：
    • ret2libc：调用 system("/bin/sh")，需要 libc 基址。
    • ROP：利用 pop rdi; ret 等 gadget 布置参数。
    • JOP / COP：较少用。

2. Canary

  • 特征：函数序言从 fs:0x28 取值放到栈上，结尾校验。
  • 泄露手法：
    • 覆盖 \x00 绕过（Canary 最低位为 \x00）。
    • 通过 printf、puts 输出栈上内容（需越界读）。
    • 逐字节爆破（适用于 fork 型服务，Canary 不变）。
  • 其他绕过：
    • 覆盖 TCB（Thread Control Block）中的 stack_guard。
    • 通过堆漏洞篡改 __stack_chk_fail 的 GOT 表项（Partial RELRO 时）。

3. PIE

  • 检测：readelf -h ./pwn | grep Type，ET_EXEC 无 PIE，ET_DYN 有 PIE。
  • 绕过：
    • 泄露任意代码段地址（如 __libc_start_main 的返回地址、函数 GOT 项）。
    • 利用相对偏移计算基址：base = leaked_addr - offset。
    • 使用 pwntools 的 ELF() 自动处理。

4. ASLR

  • 级别（/proc/sys/kernel/randomize_va_space）：
    • 0：关闭。
    • 1：栈、库、mmap 随机。
    • 2：额外开启堆随机（brk）。
  • 绕过：
    • 泄露 libc 地址（通过 GOT 泄露、printf 打印等）。
    • 部分环境可配合 ret2dlresolve 不依赖 libc 基址。

5. RELRO

  • Partial RELRO：.got 可写，.got.plt 只读（默认情况）。
    • 利用：直接修改 GOT 表项为 system 或 one_gadget。
  • Full RELRO：整个 GOT 只读，且所有函数在加载时已重定位。
    • 利用：无法改 GOT，需纯 ROP 或 hook 其他函数指针（__free_hook、__malloc_hook、_IO_2_1_stdout_ 的 vtable）。

三、组合攻击链

保护组合 攻击方案
仅 NX ret2libc（需要 libc 基址已知）
NX + ASLR 泄露 libc 地址 + ret2libc
NX + ASLR + PIE + Canary 需泄露 PIE 基址 + Canary + libc 基址，ROP
Full RELRO + NX + ASLR + PIE 只能 ROP，无法改 GOT，更依赖 gadget 链

通用步骤：

  1. 泄露 → 2. 计算基址 → 3. 构造 ROP / 劫持控制流 → 4. getshell

四、检测命令

# 1. 一键查看所有保护（推荐）
checksec --file=./pwn

# 2. 手动检查 NX
readelf -l ./pwn | grep GNU_STACK
# 若结果中没有 E（Execute）则表示 NX 开启

# 3. 检查 PIE
readelf -h ./pwn | grep Type
# ET_DYN → PIE 开启；ET_EXEC → 关闭

# 4. 检查 RELRO 类型
readelf -S ./pwn | grep relro
# .got.plt 在 RELRO 段 → Partial；.got 也在内 → Full

# 5. 查看系统 ASLR 级别
cat /proc/sys/kernel/randomize_va_space
