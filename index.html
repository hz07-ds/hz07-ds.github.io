格式化字符串漏洞

（一）格式化字符串 (Format String)

格式化字符串是一种包含普通文本和格式说明符的字符串，用于指导输出函数（如 C 语言中的 printf 系列函数）如何格式化和显示数据。

格式化字符串由两部分组成：

  1. 普通字符：直接输出到结果中的字符，原样显示。
  2. 格式说明符：以百分号 % 开始，后面跟着一个或多个字符，用于指定数据类型和输出格式。
    • 例如：%d (整数), %s (字符串), %x (十六进制), %p (地址)。

（二）格式化字符串漏洞 (Format String Vulnerability)

1. 漏洞定义

格式化字符串漏洞是一种高风险安全漏洞，通常发生在程序使用格式化字符串函数时，如 C 语言中的：

  • printf()
  • sprintf()
  • fprintf()
  • syslog()

2. 漏洞本质

这种漏洞的根本问题在于：程序将“用户可控的输入”直接当作格式化字符串（Format String）传递给了输出函数。

正常情况下，我们应该这样写：

// ✅ 安全写法：%s 是占位符，用户输入作为参数传入
char input[100];
scanf("%s", input);
printf("User input: %s\n", input);
// ❌ 危险写法：直接把用户输入当作格式化字符串处理
char input[100];
scanf("%s", input);
printf(input); // 如果用户输入了 "%x%x%s"，程序就会暴露内存！

# ret2text 与 ret2shellcode 知识总结

## 一、ret2text

### 📌 定义
返回到程序本身的代码段（text段）执行已有函数。

### 📌 利用条件
- 程序中存在可利用函数（如 win()、system()）
- 不依赖可执行栈（NX 可开）

### 📌 利用流程

  1. 找到溢出点
  2. 覆盖返回地址（RET）
  3. 将 RET 修改为目标函数地址
